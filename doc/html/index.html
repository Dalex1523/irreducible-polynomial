Irreducible Polynomial Documentation

Основной задачей данной программы является проверка полиномов на неприводимость. На вход программе подаётся список полиномов и поле для каждого из них. На выходе программы получаем информацию о неприводимости каждого из многочленов над соответствующем полем. Если полином приводим, то результат будет = 1, если неприводим, то 0.

Решение данной задачи заложено в библиотеку, в которой реализованы соответствующие классы для работы с многочленами(Polynom) и их проверка, планировщик, задача которого состоит в том, чтобы дать каждому ядру многочлен из списка на проверку и по мере завершения проверки сохранить результат и предоставить ему следующий полином для проверки.

Программа-обёртка (driver):

Программа может запускаться без параметров, с 1 или с 2 параметрами из командной строки. В случае отсутствия параметров их значения устанавливаются по умолчанию, “environ.conf” и “libirredpoly.so” соответственно

  • –c указывает путь, по которому программе следует искать файл конфигурации.
  • –l указывает путь, по которому расположен файл библиотеки.

Файл конфигурации:

В conf-файле могут быть указаны такие параметры, как количество потоков (numThread), пути к источнику, из которого берутся полиномы и к результирующему файлу (pathToSourse и pathToResult) и способ проверки полинома на неприводимость (method). Значение метода может быть установлено в 1-Matlab и 2-Berlekamp. По умолчанию используется проверка в 4 потока по методу 2 (Berlekamp) с источником “in” и результатом “out”.

В файле-источника (pathToSourse) содержатся набор многочленов.

В файле-результате (pathToResult) содержится тот же список, но перед каждым полиномом содержится 1, если он неприводим и 0, если он приводим.

Ошибки:

В случае, если в процессе работы возникает какая-то ошибка, то в программе предусмотрены следующие коды ошибок выхода:
ERROR_LIB = 1 –не удалось загрузить библиотеку для использования
ERROR_READ_CONF = 2 –не удалось прочитать файл конфигурации
ERROR_READ_POLY = 3 – не удалось прочитать полином из файла
ERROR_WRITE_POLY = 4 – не удалось записать результат в файл

Экспортируемые функции библиотеки

В библиотека имеются следующие экспортируемые функции:
create_polynom – создание полинома из строки std::string
destroy_polynom – удаление объекта типа Polynom
polynomChecker – проверка полинома на неприводимость
create_scheduler – создаёт планировщик на основании списка полиномов и числа потоков
destroy_scheduler – удаляет объект типа планировщик
